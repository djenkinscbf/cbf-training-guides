Overview

What is Google Sheets?

Google Sheets is CBF's spreadsheet tool — used for tracking budgets, monitoring project indicators, managing data, logging activities, and building reports. Like all Google Workspace apps, it saves automatically and supports real-time collaboration.

How CBF uses Google Sheets

Sheets is the go-to tool whenever structured data needs to be captured, tracked, or analysed. Common uses include project budget tracking, monitoring and evaluation (M&E) indicator logs, work plans, procurement trackers, and data submitted to donors. Sheets integrates directly with Google Docs and Slides for embedding live data in reports and presentations.

Budget tracking

Monitor expenditure against allocations across projects and cost centres

M&E indicators

Log and track project indicators, targets, and results over time

Work plans

Track activities, milestones, responsible parties, and deadlines

Stakeholder registers

Maintain contact lists, partner databases, and grantee records

Getting Started

Creating and opening spreadsheets

New sheets can be created from Google Drive or sheets.google.com. Existing Excel files can be uploaded and converted to Google Sheets format.

1
Create a new spreadsheet
  1. 1
    Go to sheets.google.com and click the blank spreadsheet thumbnail, or click + New → Google Sheets from Google Drive
  2. 2
    Click Untitled spreadsheet at the top and give it a clear name — include year, topic, and status. Example: 2026_ReefResilience_BudgetTracker
  3. 3
    Move it to the correct Shared Drive folder immediately — click the folder icon next to the file name → navigate to the right location → click Move
  4. 4
    Rename the default Sheet1 tab at the bottom — double-click it and type a descriptive name (e.g. Budget Summary, Raw Data, Jan 2026)
Tab naming matters: If your spreadsheet has multiple sheets, name every tab clearly. "Sheet1", "Sheet2", "Sheet3" is confusing for anyone else opening the file — especially during handovers or audits.
2
Upload and convert an Excel file
  1. 1
    Drag the .xlsx file into Google Drive, or click + New → File upload and select it
  2. 2
    Right-click the uploaded file → Open with → Google Sheets. This opens it as a Google Sheet while preserving the original Excel file
  3. 3
    To save it permanently as a Google Sheet: click File → Save as Google Sheets. The converted version is now fully editable in Sheets format
When to keep Excel format: If a file is regularly shared with external partners or donors who use Excel, keep the .xlsx format to avoid compatibility issues. For purely internal CBF work, convert to Google Sheets to enable full collaboration features.
Data Entry

Entering and managing data

Clean, consistent data entry is the foundation of any useful spreadsheet. These habits prevent errors and make your data reliable for reporting and analysis.

3
Enter data cleanly
  1. 1
    Click a cell and type to enter data. Press Tab to move right, Enter to move down, or arrow keys to navigate freely
  2. 2
    Keep one type of data per column — do not mix text and numbers in the same column, and do not add notes inside data cells. Use a dedicated Notes column instead
  3. 3
    Use consistent formats within each column — dates should all use the same format (e.g. DD/MM/YYYY), currency values should all be numbers without currency symbols mixed in
  4. 4
    Always include a header row in row 1 with clear column names. Freeze the header row so it stays visible when scrolling: View → Freeze → 1 row
  5. 5
    To fill a range quickly: enter a value, select the cell, then drag the small blue square (fill handle) at the bottom-right corner down or across to repeat or extend the pattern
One row = one record: Each row should represent one item — one expense, one activity, one indicator. Never merge data from two records into one row. This is the most common data entry mistake that breaks filtering and formulas.
4
Use data validation to prevent errors
Select cells→ Data→ Data validation
  1. 1
    Under Criteria, choose what type of data is allowed: a list of items, a date range, a number range, or a custom formula
  2. 2
    For a dropdown list: select Dropdown (from a range) or Dropdown and enter your options (e.g. Active, Completed, On Hold)
  3. 3
    Set the invalid data option to Show warning or Reject input to enforce the rules when someone enters data
Use dropdowns for status fields: Any column with a fixed set of options — project status, region, funding source — should use a dropdown validation. This prevents typos and ensures your data filters and pivot summaries work correctly.
Formatting

Formatting spreadsheets

Well-formatted spreadsheets are easier to read and less prone to interpretation errors. Use formatting to guide the reader's eye — not to decorate.

5
Format cells, rows, and columns
  1. 1
    Header rows: Select row 1 → make text bold, increase font size slightly, and apply a background fill colour (CBF navy #1F4A94 with white text, or light grey for simpler sheets)
  2. 2
    Column width: Double-click the column border in the header to auto-fit the width to the content — or drag to set a custom width. Right-click a column header → Resize column for a precise value
  3. 3
    Number formatting: Select cells containing currency → click Format → Number → Currency. For percentages: Format → Number → Percent
  4. 4
    Alternating row colours: Select your data range → Format → Alternating colours → choose a colour scheme. This dramatically improves readability for long data tables
  5. 5
    Borders: Select cells → click the border icon in the toolbar to add borders. Use a full outer border for the table and inner borders to separate columns
CBF colour guidance: When using colour to highlight cells (e.g. red for overspend, green for on-track), always add a legend somewhere on the sheet explaining what each colour means. Never rely on colour alone to convey meaning — some colleagues may have colour vision differences.
6
Freeze rows and columns
  1. 1
    To freeze the header row: click View → Freeze → 1 row. The first row stays visible as you scroll down through data
  2. 2
    To freeze a column (useful for ID or name columns in wide sheets): click View → Freeze → 1 column
  3. 3
    To freeze at a specific row or column: click the cell just below/to the right of where you want to freeze → View → Freeze → Up to current row/column
Always freeze header rows on any sheet with more than 10 rows of data. It is one of the simplest improvements that makes a spreadsheet significantly easier to use.
Formulas & Functions

Essential formulas for CBF work

You do not need to be a spreadsheet expert to use Google Sheets effectively at CBF. These core formulas cover the vast majority of data tasks you will encounter.

7
How formulas work
  1. 1
    Every formula starts with an = sign. Click a cell, type =, and start typing the formula name — Sheets will suggest completions
  2. 2
    Formulas reference cells by their column letter and row number — e.g. B2 means column B, row 2. A range is written as B2:B20 (all cells from B2 to B20)
  3. 3
    To copy a formula down a column: click the cell with the formula → drag the blue fill handle down. Cell references update automatically (e.g. B2 becomes B3, B4, etc.)
  4. 4
    To lock a reference so it doesn't change when copied, add a $ sign: $B$2 always refers to B2 regardless of where the formula is copied
If a formula shows an error: #REF! means a referenced cell was deleted. #VALUE! means the wrong data type was used (e.g. text where a number is expected). #DIV/0! means the formula is trying to divide by zero. Click the cell and check the formula bar to diagnose.
SUM
=SUM(B2:B20)
Adds all values in a range. The most used formula at CBF for budget totals and activity counts.
Example: =SUM(D2:D50) totals all budget figures in column D
AVERAGE
=AVERAGE(C2:C20)
Calculates the mean of a range. Useful for average spend per project or average indicator score.
Example: =AVERAGE(E2:E12) averages monthly expenditure
COUNT / COUNTA
=COUNTA(A2:A100)
COUNT counts numeric cells. COUNTA counts all non-empty cells — use COUNTA for text data like names or statuses.
Example: =COUNTA(B2:B50) counts number of completed entries
IF
=IF(D2>1000,"Over","OK")
Returns one value if a condition is true, another if false. Great for status flags and budget alerts.
Example: =IF(E2>=F2,"On Track","Behind") compares actual vs target
SUMIF
=SUMIF(A:A,"Marine",D:D)
Sums values where a condition is met. Use to total budget by project type, region, or status.
Example: totals all expenditure rows where project = "Marine Conservation"
COUNTIF
=COUNTIF(C:C,"Active")
Counts cells matching a condition. Use to count how many projects are active, complete, or on hold.
Example: =COUNTIF(B2:B50,"Complete") counts completed activities
VLOOKUP
=VLOOKUP(A2,Sheet2!A:C,2,FALSE)
Looks up a value in another table and returns a related value. Useful for pulling project names or budget codes from a reference sheet.
Example: looks up a project ID and returns the project name from a lookup table
TEXT
=TEXT(A2,"DD/MM/YYYY")
Formats a number or date as text in a specific pattern. Use to standardise date display in reports.
Example: =TEXT(TODAY(),"DD MMMM YYYY") returns today's date formatted
Data Analysis

Sorting and filtering data

Sorting and filtering let you focus on specific subsets of data without deleting anything. These are among the most used features for CBF project tracking and reporting.

8
Sort data
  1. 1
    Click any cell in the column you want to sort by → click Data → Sort sheet by column [X] A→Z or Z→A
  2. 2
    For multi-level sorting (e.g. sort by region first, then by date): click Data → Sort range → Advanced range sorting options → add sort levels
  3. 3
    Always ensure your data has a clear header row before sorting — Sheets needs to know which row is the header so it does not sort it into the data
Sort the range, not the sheet: For tables with multiple related columns, always select the full data range before sorting — not just one column. Sorting a single column shifts that column's values out of alignment with the rest of the row.
9
Add and use filters
  1. 1
    Click any cell in your data → click Data → Create a filter. Filter dropdowns (▼) appear in the header row
  2. 2
    Click a filter dropdown to filter by specific values, conditions (greater than, contains, etc.), or to search for a value
  3. 3
    Filtered rows are hidden — not deleted. Remove the filter to restore all rows: click Data → Remove filter
  4. 4
    To let each collaborator filter without affecting others' view: use Data → Filter views → Create new filter view instead — each person gets their own private filter
Use Filter Views for shared sheets: When multiple people use the same sheet, Filter Views mean your filter does not disrupt what colleagues are seeing. This is especially important for live project trackers and budget sheets.
Data Visualisation

Creating charts

Charts turn raw data into visuals that communicate clearly in reports and presentations. Charts created in Sheets can be embedded directly into Google Docs and Slides.

10
Insert a chart
  1. 1
    Select the data you want to chart — include the header row. Click Insert → Chart
  2. 2
    The Chart editor opens on the right. Under Chart type, select the most appropriate format:
    — Bar / Column chart: compare values across categories (e.g. budget by project)
    — Line chart: show trends over time (e.g. spend by month)
    — Pie chart: show proportions (e.g. budget allocation by programme area)
  3. 3
    Click the Customise tab in the Chart editor to update the chart title, axis labels, colours, and font — apply CBF primary colours (#1F4A94 navy, #418344 green) for on-brand visuals
  4. 4
    To embed the chart in a Google Doc or Slide: copy the chart (right-click → Copy) → paste it into the document. Sheets will ask if you want to link to spreadsheet — choose this so the chart updates automatically when the data changes
Link charts to data: Always link embedded charts back to the Sheets source when inserting into Docs or Slides. This means your reports always reflect the latest figures without manually updating visuals every time data changes.
Working Together

Collaborating on spreadsheets

Google Sheets supports the same real-time collaboration as Docs. These practices keep shared data sheets clean and reliable.

11
Protect ranges and sheets
  1. 1
    To prevent colleagues from accidentally editing formula cells or header rows: right-click the cells or sheet tab → Protect range or Protect sheet
  2. 2
    Set who is allowed to edit the protected range — typically only the sheet owner or data manager. Other users will see a warning if they try to edit
  3. 3
    To protect an entire sheet while allowing edits to specific input cells: protect the sheet first, then within the protection settings click Except certain cells and select the input range
Protect formula columns: In any shared budget or M&E tracker, protect the columns containing formulas. Nothing breaks a shared spreadsheet faster than a colleague accidentally overwriting a formula cell with a value.
12
Leave comments in cells
  1. 1
    Right-click a cell → Comment (or press Ctrl+Alt+M) to add a comment anchored to that specific cell
  2. 2
    A small orange triangle appears in the top-right corner of the cell — hover over it to read the comment
  3. 3
    Use @mentions in comments to tag a colleague for their attention — they receive an email notification with a link directly to the cell
  4. 4
    Resolve comments once addressed — right-click the cell → Resolve comment
Use comments, not text notes: For flagging issues, questions, or instructions in a shared spreadsheet, always use cell comments rather than typing notes directly into data cells. Notes in data cells break formulas and filters.

Best Practices

Tips for CBF staff

Habits that keep your spreadsheets reliable, shareable, and useful for the long term.

Separate raw data from summaries

Keep raw data entry on one tab and summary tables or charts on another. Never edit raw data to make summaries look right — fix the formula instead.

Protect formula cells

Any cell containing a formula that other people should not edit should be protected. This prevents accidental overwrites that silently break calculations.

Use Filter Views, not filters

On shared sheets, always use Filter Views (Data → Filter views) rather than regular filters. This lets you analyse data without disrupting your colleagues' view.

Archive final sheets to WorkDrive

When a reporting period closes, export the final spreadsheet as .xlsx or PDF and archive it to Zoho WorkDrive. Keep Google Sheets for active tracking only.

Add a legend for colour coding

If you use colour to flag statuses or thresholds, always include a small legend on the sheet. Colour-only signals are inaccessible to colleagues with colour vision differences.

Format numbers correctly from the start

Apply currency, percentage, and date formats as soon as you create a column — not after entering data. Reformatting later can cause display inconsistencies.
