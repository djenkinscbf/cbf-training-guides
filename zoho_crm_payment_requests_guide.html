Overview

What is the Payment Requests module?

The Payment Requests module is CBF's Zoho CRM workflow for submitting program or project payment requests. It keeps the request, supporting documents, approval stage, email notifications and final status in one tracked record.

One request, one record, one audit trail

Each payment request receives a unique ID, remains editable while it is in progress, sends automated notifications to the assigned approver, and stores supporting documents in a restricted folder linked to the request.

Getting Started

Accessing Zoho CRM

Use your assigned CBF Zoho account. If Zoho asks for authentication setup or blocks access, do not create a duplicate account. Contact the CBF System Administrator for access recovery.

1
Sign in to Zoho One
  1. 1
    Open Zoho One in your browser and sign in using your CBF Zoho credentials.
  2. 2
    Complete the authentication prompt if required. This may include an authenticator app or recovery verification.
  3. 3
    From the Zoho One app menu, open CRM.
  4. 4
    Confirm you are working in the correct CRM teamspace before opening the Payment Requests module.
Access issue: If you cannot log in, are blocked by authentication, or do not see CRM, stop and contact the System Administrator. Do not attempt to bypass authentication or use another person's account.
Navigation

Opening Payment Requests

The Payment Requests module is available from the CRM module menu. You can view existing requests and start a new request from the list view.

2
Open the module and start a request
  1. 1
    Inside Zoho CRM, select Modules from the top navigation.
  2. 2
    In the left module list, click Payment Requests.
  3. 3
    To begin a new submission, click Create Payment Request.
Use the left module menu to open Payment Requests. The Create Payment Request button starts a new record.
Interface note: Zoho may display the CRM interface in the classic view or newer view. Button positions may differ slightly, but the workflow and required information remain the same.
Core Workflow

Creating a payment request

A new request starts in Draft stage. Complete the required fields carefully before saving. Fields with a red indicator are required and must be completed before the request can move through approval.

3
Start the request and complete requester details
  1. 1
    Confirm the Payment Stage shows Draft. This is expected for a new request.
  2. 2
    In Requester's Name, select yourself from the lookup field.
  3. 3
    Enter your Job Title.
  4. 4
    In Vendor, select the vendor receiving payment. For consultancy self-submissions, this may be your own vendor profile.
Complete required fields first. Save keeps the request as a record so documents and workflow history can be attached.
Requester versus vendor: The requester is the person creating the request. The vendor is the party being paid. In some consultancy workflows these may be the same person, but this is not always true.
Payment Details

Completing the payment fields

The accuracy of the payment request depends on the payment type, description, amount, currency, funding source and approver. Confirm these details before submitting because they determine routing and review.

4
Enter payment type, amount, funding source and approver
  1. 1
    Select the correct Payment Type. For the demonstrated consultancy workflow, the payment type was Consultancy.
  2. 2
    In Payment Description, enter a clear description such as Payment June 2026 or Invoice [number] - [period].
  3. 3
    Enter the Payment Amount and verify the Payment Currency. Use USD only when that is the correct payment currency for the request.
  4. 4
    Select the correct Funding Source and, if required, the Activity Line / Project.
  5. 5
    Under Approval, select the assigned Approver. This person receives the automated review notification.
Funding source and approver are workflow-critical fields. Use the project and reviewer assigned to your payment request.
Do not copy example values: In the demonstration, the request used a specific funding source and approver for that user. Your request may require a different funding source, project line and approver.

Payment Type

Controls how the request is classified. Select the option that matches the payment being requested.

Funding Source

Connects the payment to the correct project or funding stream. Confirm this before saving.

Approver

Routes the request to the person responsible for review and approval.

Description

Use enough detail for Finance and the approver to understand the payment period and purpose.

5
Save the draft record
  1. 1
    After completing the core fields, click Save. This creates the payment request record.
  2. 2
    Wait for Zoho to open the saved record. The system will assign a unique payment request ID, such as PT893.
  3. 3
    Review the record to confirm the stage, requester, vendor, payment type and amount are correct.
Successful save: Once the record is saved, it can generate the linked document folder and begin showing activity in the record timeline.
Supporting Documents

Uploading documents

Supporting documentation should be attached through the folder generated from the payment request. This keeps the documents tied to the request and limits visibility to the appropriate approver and Finance users.

6
Open the restricted folder and upload documents
  1. 1
    From the saved payment request record, open the linked document folder generated for that request.
  2. 2
    Upload the required supporting files, such as the invoice, report, deliverable evidence, or other documentation requested by Finance or the approver.
  3. 3
    Wait until Zoho confirms the upload is complete before closing the folder or returning to CRM.
  4. 4
    Return to the payment request record and click Save again if required to finalize the submission for approval.
Upload documents to the request-specific folder. Wait for the upload confirmation before returning to CRM.
Attachments by payment type: The module supports several payment types, and each has its own required attachments, such as the invoice, an approved work plan, or approved reports. Upload the documents that apply to your payment type.
Folder permissions: The generated folder is restricted. It is intended to be visible only to the designated approver and Finance team members involved in the payment workflow.
Status Tracking

Tracking approval and completion

After submission, Zoho updates the request stage and records key activity in the Timeline. Use the request ID when following up with the approver or Finance.

7
Review the saved record and current stage
  1. 1
    Open the payment request record from the list view or from your direct link.
  2. 2
    Check the unique Payment Request ID. Use this ID in follow-ups.
  3. 3
    Review the Payment Stage. A submitted request may show Awaiting Approval.
  4. 4
    If the record is still editable, use Edit only to correct legitimate mistakes before final closure.
The record view shows the payment request ID, the current stage, and the Submit for Approval action that routes it to the approver.
8
Use Timeline to follow the approval trail
  1. 1
    Open the saved payment request.
  2. 2
    Click the Timeline tab.
  3. 3
    Review the timeline history for automated emails, stage changes and workflow actions.
  4. 4
    Watch your email for notifications when the request is approved, completed, or closed by Finance.
The Timeline records automated notifications and stage changes, including the transition from Draft to Awaiting Approval.
What happens next: The approver receives an automated email notification. After Finance completes and closes the request, the record becomes final and should not be edited.
CBF Rules

Important workflow and data rules

Payment requests create financial records. Take extra care with accuracy, attachments and permissions.

Non-negotiable rules
✕Do not use another person's Zoho account. Payment request activity is tracked and should reflect the correct requester.
✕Do not guess the funding source or approver. Incorrect routing can delay payment and create cleanup work for Finance.
✕Do not upload payment documents to random folders. Use the restricted folder generated from the payment request unless instructed otherwise.
✕Do not edit closed records. Once Finance closes a request, treat the record as final and contact Finance or the System Administrator if a correction is needed.
✕Do not attach unnecessary sensitive information. Upload only the documentation required to support the payment request.
Best Practices

Tips for smooth payment requests

Small checks before submission reduce delays and follow-up emails.

Check required fields first

Red indicators mean the field is required. Complete those fields before trying to save or submit.

Use a clear description

Include payment period, invoice number or purpose so the approver understands the request quickly.

Upload before follow-up

Make sure the invoice and supporting documents are uploaded before asking the approver to review.

Use the request ID

When following up, reference the payment request ID rather than sending vague descriptions.

Check Timeline

Before asking for an update, open Timeline to see the latest automated email and stage change.

Keep documents in the secure folder

The generated folder is the safest place for supporting files because access follows the request workflow.

Need Help?

Getting support

For login issues, missing module access, broken request links, folder permission errors or workflow questions, contact the CBF System Administrator. For payment policy, funding source or approver confirmation, contact the assigned supervisor, program contact or Finance team.

Contact & Resources

Use support when you are blocked, when the correct funding source or approver is unclear, or when a submitted request needs correction after closure.
