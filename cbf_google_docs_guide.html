Overview

What is Google Docs?

Google Docs is CBF's primary tool for creating and collaborating on written documents — reports, proposals, meeting notes, policies, and more. It works entirely in the browser, saves automatically, and lets multiple people work on the same document at the same time.

How CBF uses Google Docs

Google Docs is used for drafting and reviewing project reports, writing meeting notes, developing policies and procedures, and creating any document that requires input from more than one person. Because it saves every change automatically, there is no risk of losing work — and every version is accessible through version history.

Getting Started

Creating and opening documents

You can create a new document from Google Drive, from docs.google.com, or directly from Gmail. Opening an existing document is as simple as clicking it in Drive.

1
Create a new document
  1. 1
    Go to docs.google.com and click the blank document thumbnail, or click + New → Google Docs from inside Google Drive
  2. 2
    Click Untitled document at the top-left and type a clear, descriptive name. Example: 2026_Q2_MarineConservation_Report_DRAFT
  3. 3
    The document is saved automatically in My Drive by default — move it to the correct Shared Drive folder straight away so it is accessible to your team
  4. 4
    To move it: click the folder icon next to the document title at the top → navigate to the correct Shared Drive location → click Move
Name before you share: Always rename a document before sharing it. Collaborators seeing "Untitled document" have no idea what they are looking at. Include the year, topic, and status (DRAFT / FINAL) in the name.
2
Create from a template
  1. 1
    Go to docs.google.com → click Template gallery (top-right of the homepage) to browse available templates
  2. 2
    CBF templates — such as meeting note formats, report structures, or letter templates — may be stored in your team's Shared Drive. Check there before creating from scratch
  3. 3
    To use a template from Drive: open the template file → click File → Make a copy. This creates a new editable version and leaves the original template intact
Always copy, never edit the original: When using a shared template, always make a copy first. Editing the original template affects everyone who uses it.
Document Formatting

Formatting your document

Well-formatted documents are easier to read and reflect CBF's professional standards. Use the built-in heading styles and formatting tools — avoid manual formatting with bold text and font size changes alone.

3
Apply heading styles
  1. 1
    Click on the text you want to format as a heading. In the toolbar, click the style dropdown that says Normal text
  2. 2
    Select the appropriate heading level:
    — Heading 1: main section titles (used once or twice per document)
    — Heading 2: subsection titles
    — Heading 3: sub-subsections or minor headings
  3. 3
    Using heading styles (not just bold text) enables the Document Outline panel, allows automatic Table of Contents generation, and makes the document accessible to screen readers
Why this matters at CBF: Consistent heading use means anyone can navigate long reports instantly using the outline panel. It also ensures documents are accessible to colleagues using assistive technologies.
4
Set page layout and document settings
  1. 2
    Go to File → Page setup to set paper size (A4 for CBF documents), orientation, and margins
  2. 2
    To set the default font: format any text in your chosen font → Format → Paragraph styles → Normal text → Update 'Normal text' to match
  3. 3
    CBF documents should use Outfit or Arial as the body font, in keeping with the CBF Brand Toolkit typography guidelines
  4. 4
    To add page numbers: Insert → Headers & footers → Page numbers — choose position (top or bottom) and alignment
CBF font guidance: The official CBF typeface is Outfit. If Outfit is not available, Arial is the recommended substitute. Avoid decorative or non-standard fonts in official CBF documents.
5
Insert a table of contents
  1. 1
    Place your cursor where you want the table of contents to appear (typically after the title page or introduction)
  2. 2
    Click Insert → Table of contents → choose a style (with page numbers, or with blue links)
  3. 3
    The table of contents is generated automatically from your Heading styles. If you add or move sections, click the refresh icon on the table to update it
For long reports: Always include a table of contents in documents longer than 5 pages. It saves readers significant time navigating CBF reports, proposals, and policy documents.
Editing Modes

The three editing modes

Google Docs has three modes that control how your changes appear. Understanding when to use each one is essential for clean collaboration at CBF.

Editing
Your changes apply directly and immediately to the document
Use when: you own the document or have been given direct editing authority
Suggesting
Your changes appear as tracked suggestions for the owner to accept or reject
Use when: reviewing or contributing to someone else's document
Viewing
Read-only mode — no changes can be made to the document content
Use when: reading a final document or reviewing without any intent to edit
6
Switch between editing modes
  1. 1
    Click the pencil icon in the top-right area of the document (next to the Share button) — this shows your current mode
  2. 2
    Click it to open the mode dropdown and select Editing, Suggesting, or Viewing
  3. 3
    In Suggesting mode, your additions appear in green underlined text and deletions appear as strikethrough — each suggestion has an Accept ✓ or Reject ✗ button for the document owner
CBF default when reviewing: Always switch to Suggesting mode before editing a document you did not create. Making direct edits to someone else's draft without using suggestions can overwrite their work without a trace.
Feedback & Review

Comments and suggestions

Comments let you leave feedback, ask questions, and tag colleagues on specific parts of a document — without altering the document content itself.

7
Add a comment
  1. 1
    Highlight the text you want to comment on, then click the comment bubble icon that appears in the right margin — or press Ctrl+Alt+M (PC) / ⌘+Option+M (Mac)
  2. 2
    Type your comment in the box. To tag a specific colleague, type @ followed by their name — they receive an email notification with a direct link to the comment
  3. 3
    Click Comment to post. Your comment appears in the margin anchored to the highlighted text
  4. 4
    To reply to an existing comment: click the comment → type in the Reply field. This creates a thread, keeping the discussion together
Use @mentions for action items: When a comment requires a specific person to act, always @mention them. This ensures they receive a notification and the responsibility is clearly assigned within the document itself.
8
Resolve and manage comments
  1. 1
    Once a comment has been addressed, click Resolve (the tick icon on the comment) — the comment disappears from the main view but is preserved in the comment history
  2. 2
    To view all comments including resolved ones: click Open comment history (the speech bubble icon top-right of the document)
  3. 3
    To reopen a resolved comment: find it in comment history and click Re-open
Before finalising a document: Resolve all open comments. A document shared externally or archived to Zoho WorkDrive should have no unresolved comments — they signal that the document is not yet complete.
Working Together

Real-time collaboration

Multiple CBF staff can work in the same document simultaneously. Here is how to collaborate effectively without creating confusion.

9
Collaborate in real time
  1. 1
    When multiple people have the document open, you will see their coloured cursors moving through the document in real time — each person's name appears beside their cursor
  2. 2
    To avoid confusion when multiple people are editing simultaneously, use the document chat: click the chat bubble icon (top-right) to message collaborators currently in the document without leaving it
  3. 3
    If you need to work on the same section as someone else, coordinate via chat or comments first — simultaneous edits to the same paragraph can sometimes result in conflicts
Version history is your safety net: If real-time collaboration creates errors, go to File → Version history → See version history to find a clean earlier version and restore it. Google Docs saves a version every few minutes automatically.
10
Accept and reject suggestions
  1. 1
    When a colleague has made suggestions (in Suggesting mode), they appear as green highlighted changes in the document
  2. 2
    Click the tick ✓ on a suggestion to accept it — the change is incorporated into the document text
  3. 3
    Click the X on a suggestion to reject it — the original text is restored
  4. 4
    To accept or reject all suggestions at once: click Tools → Review suggested edits → Accept all or Reject all
Review before accepting all: Always read through suggestions individually before using "Accept all" — especially on important CBF reports or policy documents where a single misplaced word can change meaning significantly.
Document Structure

Using the outline, bookmarks & links

For longer CBF documents — project reports, annual reviews, grant proposals — these tools keep navigation clean and professional.

11
Use the document outline panel
  1. 1
    Click View → Show outline to open the outline panel on the left side of the screen
  2. 2
    The outline displays all Heading 1, 2, and 3 text as a clickable navigation tree — click any heading to jump directly to that section
  3. 3
    This is especially useful when reviewing long reports — share the document with the outline open by default so reviewers can navigate without scrolling
12
Insert links and cross-references
  1. 1
    To insert a hyperlink: highlight text → press Ctrl+K (PC) / ⌘+K (Mac) → paste a URL or search for a heading within the document to link to
  2. 2
    To link to another section within the same document: click Insert → Bookmark to place a bookmark, then use Ctrl+K and select the bookmark as the link destination
  3. 3
    To link to a Google Drive file: paste the Drive file URL, or use Ctrl+K → type the file name — Docs will find it automatically if you have access
Link to Drive files, not attachments: When referencing supporting documents in a report, link to the Google Drive file rather than attaching a copy. This keeps everything up to date and reduces duplication.
Exporting

Exporting and downloading documents

When a document is complete and ready to be shared externally, submitted, or archived to Zoho WorkDrive, export it to the appropriate format.

13
Export a document
File→ Download→ Select format
FormatUse at CBF
.pdf Final versions for external sharing, submission, or archiving to Zoho WorkDrive. Layout is locked and cannot be edited by recipients
.docx When sharing with external partners, donors, or consultants who use Microsoft Word
.odt Open-format alternative to Word — use when a partner specifically requests an open document format
.txt Plain text only — use when content needs to be pasted into another system without any formatting
.html For web publishing — use only when content is being added directly to a website or Google Sites
Archive as PDF to WorkDrive: When a document is finalised and approved, export as PDF and upload to the correct folder in Zoho WorkDrive. PDF preserves the layout exactly and cannot be accidentally edited after archiving.

Best Practices

Tips for CBF staff

Habits that make your Google Docs cleaner, more professional, and easier for the whole team to work with.

Use Suggesting mode by default

When opening a shared document to review it, switch to Suggesting mode immediately. This protects the original content and creates a clear review trail.

Include a document header

Start every formal CBF document with a header block: document title, date, author name, version number (v0.1 for draft, v1.0 for final), and status (DRAFT / FOR REVIEW / FINAL).

Tag people, not just mention them

In comments, use @name to tag a specific colleague rather than writing their name in plain text. This sends them a direct notification and keeps accountability clear.

Clear comments before archiving

Before exporting a final document to PDF and uploading to Zoho WorkDrive, resolve all comments and accept or reject all suggestions. A clean document signals it is truly final.

Use Find & Replace for consistency

Press Ctrl+H to open Find & Replace — useful for updating project names, correcting repeated errors, or standardising terminology across a long document quickly.

Keep one document per topic

Avoid cramming multiple unrelated topics into one document. Separate documents are easier to share, version, and archive. Link related documents to each other instead of combining them.
